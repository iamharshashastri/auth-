Overview

UltiAuth provides drop-in authentication for multi-tenant applications. Each app you create gets isolated user pools, configurable OAuth providers, and JWT-based sessions.

All API endpoints are available at:

https://lqkpxervwakucksrnoss.supabase.co/functions/v1/

How it works

  • Create an app in the UltiAuth dashboard and get a publishable key
  • Embed the auth UI at /v1/auth/?key=YOUR_KEY&redirect_uri=YOUR_CALLBACK
  • After sign-in, users are redirected back with a JWT token
  • Your app verifies the JWT and uses the User API on behalf of the user

Quick Start

JavaScript
HTML
// 1. Redirect to UltiAuth login page
const loginUrl = `https://lqkpxervwakucksrnoss.supabase.co/v1/auth/`
  + `?key=pk_YOUR_PUBLISHABLE_KEY`
  + `&redirect_uri=${encodeURIComponent(window.location.origin + '/callback')}`
  + `&state=random_csrf_token`;

window.location.href = loginUrl;

// 2. On your callback page, grab the token
const token = new URLSearchParams(window.location.search).get('token');

// 3. Use the token to call the User API
const res = await fetch('https://lqkpxervwakucksrnoss.supabase.co/functions/v1/user-api/me', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const user = await res.json();

Authentication

The User API uses Bearer token authentication. Include the JWT token issued by UltiAuth in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

JWT Payload

UltiAuth JWTs contain the following claims:

ClaimTypeDescription
substringUser ID (UUID)
emailstringUser email address
app_idstringApp ID the user belongs to
iatnumberIssued at (Unix timestamp)
expnumberExpiry (Unix timestamp, 7 days)

Verifying Tokens

JavaScript
curl
// Verify using the /verify endpoint
const res = await fetch(
  `https://lqkpxervwakucksrnoss.supabase.co/functions/v1/auth-handler/verify?token=${token}`
);
const { valid, payload } = await res.json();
if (valid) {
  console.log('User ID:', payload.sub);
  console.log('Email:', payload.email);
}

Error Handling

All errors return a JSON body with an error field and appropriate HTTP status codes.

{
  "error": "Invalid credentials",
  "code": "AUTH_INVALID_CREDENTIALS"
}
StatusMeaning
200Success
400Bad request – missing or invalid parameters
401Unauthorized – missing or invalid JWT
403Forbidden – insufficient permissions
404Not found
409Conflict – e.g. email already exists
500Internal server error

GET /auth-config

Returns the public configuration for an app given its publishable key. Used by the auth UI to determine which providers to show.

GET /auth-config?key={publishable_key} Get app configuration

Query Parameters

ParameterTypeRequiredDescription
keystringrequiredApp publishable key (starts with pk_)

Response

  "name": "My App",

  "provider_configs": [
    { "provider": "google", "client_id": "..." }
  ],
  "redirect_urls": ["https://myapp.com/callback"]
}

POST /auth-handler/signup

      
Create a new end-user account in an app. The password is bcrypt-hashed before storage.

      

        

          POST
          /auth-handler/signup
          Create user account
        

        

          
Request Body

          

            
FieldTypeRequiredDescription

            
              
emailstringrequiredUser email address

              
passwordstringrequiredPassword (min 8 chars)

              
app_keystringrequiredApp publishable key

              
namestringoptionalDisplay name

            
          

          
Response

          
{

    "id": "uuid",
    "email": "user@example.com",
    "name": null,

DELETE /user-api/me

      
Permanently delete the authenticated user's account and all associated data.

      

        

          DELETE
          /user-api/me
          Delete user account
        

        

          
No request body required. Returns 204 No Content on success. This action is irreversible.
